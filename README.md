# Simple Wordbook

> ✅ **当前版本**：v1.5.0 | 最低兼容 Obsidian v1.0.0（在 v1.12.7 上验证通过）  
> 📦 **插件 ID**：`simple-wordbook`  
> 📌 **项目结构**：纯 JavaScript 编写，无构建工具，`main.js` 即源码  
> 💡 **灵感来源**：[obsidian-language-learner](https://github.com/guopenghui/obsidian-language-learner) 和 [HiWords](https://github.com/CatMuse/HiWords)

---

## 📖 概述

**Simple Wordbook** 是一款为 [Obsidian](https://obsidian.md/) 设计的单词/短语高亮与学习管理插件。  
它能自动高亮您笔记中出现在自定义词库里的单词，并提供侧边栏、查词面板、掌握状态追踪、AI 辅助查询等完整学习工具链。

---

## ✨ 功能特性

### 📚 单词本管理

- 支持多个 `.json` 格式的单词本文件，可自由添加、移除、启用/禁用。
- 支持拖拽排序，调整词库优先级。
- 支持**只读模式**，防止误修改。
- 支持单词的**音标**、**释义**、**别名**、**颜色标签**（红/橙/黄/绿/蓝/紫）。
- 释义支持完整的 Markdown 格式，可包含表格、列表、内部链接等。

### 🎯 文档高亮

- 在**阅读模式**、**编辑模式**以及 **PDF** 中自动高亮词库内的单词。
- 支持**别名匹配**，自动高亮同一单词的不同形式。
- 高亮样式高度可定制：
    - 背景色（可跟随卡片颜色或自定义）
    - 下划线样式（无 / 实线 / 虚线 / 点状 / 波浪 / 双线）
    - 粗体
- 支持**路径范围过滤**，仅高亮指定文件夹或文件（包含/排除模式）。

### 📖 侧边栏卡片

- 自动扫描当前文档，提取所有匹配的单词并展示为卡片。
- 卡片显示：单词、音标、释义、来源词库、颜色标签。
- 支持 **「学习 / 记住 / 忽略」** 三种视图分类。
- 支持**掌握/忽略标记**，可选择**全局统一**或**按词源独立**两种管理模式。
- 卡片内容区域独立滚动，支持多段落释义标签切换。
- 支持卡片搜索过滤。

### 🔍 查词面板

- **本地查询**：在已启用的词库中快速检索单词/别名，支持智能、精确、前缀、包含、模糊等多种匹配模式。
- **AI 查询**：调用大语言模型（OpenAI、DeepSeek、智谱 GLM、通义千问、Ollama 等）获取详细释义。
- 支持**自定义提示词**（可添加多套提示词，在面板中切换使用）。
- 查询结果可直接保存为单词卡，自动提取音标。
- 三种**回车模式**：仅本地 / 仅 AI / 本地优先。
- 两种**清除按钮**：清空输入框 / 清空输出框。

### 🤖 AI 配置

- 支持多种服务商：OpenAI、DeepSeek、智谱 GLM、通义千问、Ollama（本地）。
- 支持自定义 API 地址、密钥、模型名称。
- 默认提示词可自由编辑（支持 `{word}` 占位符）。
- 自定义提示词管理（添加/编辑/删除）。
- **设置快捷键按钮**：在“添加自定义提示词”按钮右侧，点击“设置快捷键”按钮，会自动跳转到 Obsidian 快捷键设置页，并自动过滤显示本插件的所有命令，方便您快速为各个提示词命令绑定热键。
- **测试连接**功能，快速验证配置是否正确。

### 🖱️ 悬浮预览

- 鼠标悬停在高亮单词上，弹出释义弹窗。
- 弹窗显示音标、完整释义、多词源来源。
- 弹窗内可直接标记掌握/忽略，或定位到侧边栏。
- 支持**模糊释义**模式（默认模糊，悬停清晰）。

### 🔊 发音功能

- 点击单词即可播放 TTS 语音。
- 支持美音/英音切换。
- 可自定义 TTS URL 模板（支持 `{{word}}`、`{{type}}`、`{{accent}}` 占位符）。

### ⌨️ 命令与快捷键

插件注册了以下命令（可在 Obsidian 设置 → 快捷键中自定义热键）：

| 命令名称     | 说明              |
| -------- | --------------- |
| `打开侧边栏`  | 激活侧边栏视图         |
| `打开查词面板` | 打开查词面板并聚焦输入框    |
| `添加单词`   | 打开添加单词模态框       |
| `刷新单词本`  | 重新加载所有词库文件并刷新高亮 |
| `打开设置`   | 跳转到插件设置页面       |

**动态生成的命令**：

在 `AI 配置 → 自定义提示词` 中，每添加一个自定义提示词，插件会自动生成一个独立的命令：

| 命令格式 | 说明 |
| :--- | :--- |
| `查词提示词：{提示词名}` | 选中单词后执行该命令，将以对应的自定义提示词进行 AI 查询 |

这些命令会**自动注册**，用户可在 Obsidian 设置 → 快捷键中为每个命令绑定独立的热键（如 `Ctrl+Shift+1`、`Ctrl+Shift+2` 等），实现一键以特定提示词查词。

- 添加/删除/重命名提示词后，命令列表**自动刷新**，无需重启 Obsidian。
- **快捷键永久绑定**：只要不修改提示词名称，绑定的快捷键永久有效。
- 删除提示词后，对应的命令会自动移除，快捷键会进入“悬空”状态（不会报错）；若重新添加同名提示词，快捷键**自动恢复**。

**编辑器右键菜单**：

- 选中文本后，右键菜单出现「添加单词/短语」和「查词：…」选项。
- 「查词」会弹出子菜单，让您选择**默认**或**自定义提示词名**，直接执行 AI 查询。

### 🎨 自定义样式（CSS 片段）

您可以通过 CSS 片段覆盖插件默认样式。以下示例让侧边栏卡片内的高亮词背景色跟随卡片颜色，且透明度为 10%（插件已内置，可在 `styles.css` 中查看）：

```css
.word-card .simple-wordbook-highlight,
.word-card-content .simple-wordbook-highlight,
.lookup-result .word-card .simple-wordbook-highlight {
  background-color: color-mix(in srgb, var(--card-color) 10%, transparent) !important;
  text-decoration: none !important;
  border-radius: 6px !important;
  padding: 0 2px !important;
}
```

更多定制可参考插件 `styles.css` 中的命名空间。

## 🚀 安装方法

### 手动安装

1. 下载 `main.js`、`manifest.json`、`styles.css`。
2. 将其放入你的 vault 的 `.obsidian/plugins/simple-wordbook/` 目录。
3. 在 Obsidian 设置中启用插件。

### 通过 BRAT 安装（测试版推荐）

1. 安装 [BRAT](https://obsidian.md/plugins?id=obsidian42-brat) 插件并启用。
2. 在 BRAT 设置中点击 **Add Beta Plugin**。
3. 输入仓库地址：`https://github.com/Bin-T/obsidian-simple-wordbook`。
4. 点击 **Add Plugin**，然后回到社区插件列表启用即可。

## 🎮 使用指南

### 1. 添加单词本

1. 在 vault 中新建一个空白 `.json` 文件，或导入单词格式如下的 `.json` 文件词库（词库格式为 JSON 数组，每个单词对象需包含 `word` 字段，可选 `aliases`、`definition`、`phonetic`、`color` 等。）：

```json
[
  {
    "word": "abandon",
    "aliases": ["abandoned", "abandoning"],
    "phonetic": "英 /əˈbændən/ 美 /əˈbændən/",
    "definition": "**释义**\n 放弃，遗弃\n\n **例句**\nHe abandoned his car in the snow.",
    "color": "red"
  }
]
```

2. 打开 **设置 → Simple Wordbook → 文件管理**，点击 **“添加单词本文件”**，选择你的 `.json` 文件。
3. 文件会出现在列表中，可通过开关启用/禁用，或设为只读。


### 2. 侧边栏使用

- 点击左侧 Ribbon 中的 **📖** 图标或命令打开侧边栏。
- 侧边栏会自动显示当前文档中匹配的单词卡片。
- 点击卡片上的 **😊/😐** 标记掌握/取消掌握。
- 点击 **👁/🚫** 标记忽略/取消忽略。
- 点击单词可发音（需配置 TTS）。
- 对未设置只读词源的单词卡片可右键 **编辑/删除** 单词。

### 3. 查词面板使用

- 点击左侧 Ribbon 中的 **🔍** 图标或命令打开查词面板。
- 输入单词后，点击 **“本地查询”** 在词库中检索。
- 点击 **“AI 查询”** 调用 AI 获取详细释义。
- 在 **“提示词”** 下拉菜单中切换提示词。
- 查询结果可直接点击 **“保存单词”** 存入词库。

#### 回车模式

在 **设置 → 常规设置 → 查词面板 → 回车模式** 中可选：

- **仅本地**：回车只查本地词库。
- **仅 AI**：回车只查 AI。
- **本地优先**：先本地，未找到再调用 AI。

#### 本地查询模式

在 **设置 → 常规设置 → 查词面板 → 本地查询模式** 中可选：

- **智能匹配 (综合排序)**
- **精准匹配 (仅完全一致)**
- **前缀匹配**
- **包含匹配**
- **模糊匹配 (允许拼写错误)**

#### 最大结果数

在 **设置 → 常规设置 → 查词面板 → 最大结果数** 中修改设置，本地查询最多返回的 **结果数量(1-100)**。

### 4. 右键菜单

- 在编辑器中选中文本，右键 → **“添加单词/短语”** 快速添加单词。
- 选中文本，右键 → **“查词：xxx”** 打开查词面板并自动查询（始终走 AI）。

### 5. 悬停预览

- 将鼠标悬停在任何高亮单词上，会自动弹出释义弹窗。
- 弹窗内可切换多词源、标记掌握、定位侧边栏。

### 6. 高亮配置

在 **设置 → 常规设置 → 高亮与预览** 中可调整：

- **启用/禁用自动高亮。**
- **高亮颜色**：跟随卡片颜色或自定义。
- **下划线样式**：无/实线/虚线/点状/波浪/双线。
- **粗体**
- **悬停预览**
- **模糊释义**

## ❓ 常见问题（FAQ）

**Q: 高亮不生效？**  
A: 请检查设置中“启用自动高亮”是否开启；检查词库文件是否启用；检查当前文件是否在“高亮范围”内。

**Q: 侧边栏显示为空？**  
A: 确保当前文档包含词库中的单词，且这些单词未被标记为“忽略”。如果启用了“侧边栏范围过滤”，请确认当前文件在范围内。

**Q: AI 查询失败？**  
A: 请检查 API 地址、密钥、模型名称是否正确，网络是否通畅。您可以在设置中使用“测试连接”验证。

**Q: 如何在 PDF 中高亮？**  
A: 默认支持 PDF 高亮。确保已启用自动高亮，并等待 PDF 渲染完成。

**Q: 掌握状态丢失？**  
A: 请检查“掌握状态文件”路径是否正确，是否可写。若切换了掌握模式（全局/按词源），插件会自动迁移数据。

**Q: 悬停预览/单词卡片中的标签页是如何拆分的以及标签页的名字是怎么定义的？**
A: 
1. 插件使用 `---` （三个连续的短横线）作为分隔符，将释义文本切分成多个部分，每个部分独立成为一个标签页。
2. 插件会扫描上述每个部分**开头的加粗文本**（`**加粗文字**`），将其提取为标签页显示的名称。

**示例：** 如果你在**编辑单词/短语**模态框的释义框里如下写，就会生成 3 个标签页：

```text
**释义**
放弃，遗弃；丢弃

---

**常用例句**
He abandoned his car in the snow.
他在雪地里丢弃了他的车。

---

**相关短语**
abandon hope 放弃希望
```

3. 如果某个部分没有以 `**标题**` 开头，插件会自动分配默认名称：

	- 第一个部分（无论有没有标题）→ 默认名为 ** `释义` **
	- 后续的部分 → 默认名为 ** `内容 2` **、** `内容 3` **……

**Q: 如何快速为自定义提示词绑定快捷键？**  
A: 在 `AI 配置 → 自定义提示词` 区域，点击“设置快捷键”按钮，会自动跳转到 Obsidian 快捷键设置页，并自动搜索过滤本插件的所有命令。您可以为每个“查词提示词：xxx”命令绑定独立的热键，实现一键查词。
